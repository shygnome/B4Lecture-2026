# Exercise 3: TrackID3x3 を用いたアブレーションスタディ

## 1. 目的と実験条件
本課題では、人物検出器と追跡器の組み合わせを変更し，トラッキング性能がどのように変化するかを確認する。


## 2. 実験結果
各条件における評価指標の比較結果を以下の表に示す。（各指標の最高値は太字で表記）

| Detector | Tracker | TI-HOTA (%) ↑ | TI-DetA (%) ↑ | TI-AssA (%) ↑ |
|:---|:---|---:|---:|---:|
| YOLO11n | ByteTrack | 25.82 | 23.73 | 28.10 |
| YOLO11n | BoT-SORT | 16.09 | 15.70 | 16.50 |
| YOLO11m | ByteTrack | 53.68 | 50.67 | 56.87 |
| YOLO11m | BoT-SORT | 53.75 | 51.36 | 56.24 |
| YOLO11x | ByteTrack | **74.40** | **73.73** | **75.08** |
| YOLO11x | BoT-SORT | 74.08 | 73.33 | 74.84 |


## 3. 考察

上記の結果より、以下の4点が確認できた。

1. **最も良かった組み合わせ**
   `YOLO11x` と `ByteTrack` の組み合わせが、すべての評価指標において最も高い性能を示した。

2. **検出器のサイズによる性能変化**
   検出器のモデルサイズを大きくする（n → m → x）につれて、すべての指標が改善した。
   追跡器が同じでも、検出器の性能向上によって対応付け性能の評価指標である TI-AssA も大幅に向上した。

3. **追跡器の変更による指標の変化**
   検出器が `YOLO11m` や `YOLO11x` の場合は、追跡器を `ByteTrack` から `BoT-SORT` に変更しても、TI-HOTA と TI-DetA はほとんど変わらなかった。
   特に、TI-AssA は `ByteTrack` よりも `BoT-SORT` の方がわずかに低い値を示した。
   一方、検出器が `YOLO11n` の場合は、追跡器を `ByteTrack` から `BoT-SORT` に変更することで、すべての指標が大幅に低下した。
   これは、検出器の精度が悪いため `BoT-SORT` が見た目の特徴を正しく抽出できなかったことによる可能性がある。

4. **検出性能とID維持性能の課題分析**
   総合的な性能評価指標である TI-HOTA は、TI-DetA と TI-AssA の相乗平均によって計算されるため、 TI-HOTA 向上のためには両者の差を小さく保ちつつ両方を高める必要がある。
   実験結果から、いずれの検出器の場合も、TI-DetA よりも TI-AssA の方が高い値を示していることがわかる。
   つまり、ID維持性能よりも検出性能に課題がある。
